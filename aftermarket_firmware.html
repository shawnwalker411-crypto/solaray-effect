Why Aftermarket Firmware?

Stock Bitmain firmware doesn't allow you to underclock miners enough to run on 120V power. Aftermarket firmware unlocks this ability, letting you:

  • Lower frequency/voltage to reduce power draw
  • Run industrial ASICs on standard 120V/15A outlets
  • Fine-tune performance vs power consumption
  • Often improve efficiency (J/TH)

⚠️ Aftermarket firmware may void your warranty. Always research compatibility with your specific miner model and control board before flashing.

Shutdown, Restart & Scheduling

If you mine on solar or time-of-use electricity rates, you need to stop and start miners on a schedule. How that works depends entirely on your firmware. Some make it easy. Others will brick your remote access if you use the wrong command.

⚠ The Poweroff Trap: Running poweroff or shutdown over SSH kills the miner's network stack. The miner is now off and unreachable — nothing over the network can turn it back on. You must physically power cycle it or use a smart PDU. This applies to ALL firmware.

Stock Firmware — SSH Is Locked

Bitmain locked SSH access on S19-series miners starting in 2019. If you're on stock firmware, you cannot use any SSH commands — no poweroff, no reboot, no cron scheduling. Your only remote option is the miner's web interface (for monitoring) and a smart PDU (for power cycling). This is the #1 reason home miners switch to aftermarket firmware.

Braiins OS+ — The Most Flexible

bosminer stop pauses mining without shutting down the miner — your SSH session stays alive and the miner stays on the network. bosminer start resumes mining. This is the safest software method for solar schedules. Cron jobs work reliably for automated stop/start. Known issue: pool settings may reset after a full restart — verify your pool config after any reboot.

LuxOS — Use PDU, Not SSH

LuxOS supports remote management through its API and web UI, but poweroff over SSH leaves the miner completely stuck off with no remote recovery path. If you're on LuxOS and need to shut down on a schedule, use a smart PDU to cut and restore power rather than any SSH shutdown command. See our 240V Power Control guide for recommended hardware.

For Loki Kit Users

If you're running a Loki Kit setup on 120V, you'll need firmware that supports:

  • Fixed frequency mode - Set a specific frequency to keep power under 1200W
  • Power limit - Some firmware lets you set a wattage cap
  • Hash board spoofing - The Loki Kit tricks the miner into thinking all boards are present

Most Loki builds use LuxOS or VNish. Start at the lowest frequency setting, then slowly increase until you hit ~1200W at the wall.
